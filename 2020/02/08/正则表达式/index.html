正则表达式

发表于 2020-02-08 更新于 2021-03-01 分类于 Java 热度： ℃ 评论：
本文字数： 872 阅读时长 ≈ 1 分钟
描述的是一个规则，匹配一类字符串

1. 转义字符

\n 换行符
\t 制表符
\\ \
\^,\$,\.,\(,\),\{,\},\?,\+,\-,\[,\],\| 匹配字符本身

2. 标准字符

\d 0-9数字
\w 字母，数字，下划线 A-Z,a-z,0-9
\s 空格，制表符，换行符等空白字符的其中任意一个
. 任意字符

3. [] 内任意字符

[ab5@] 匹配 a\b\c\5\@
[^abc] a,b,c以外的任意字符
[f-k] f-k之间的任意字母
[^A-F0-3] A-F,0-3之外的任意字符

包含在中括号中的特殊符号会失去特殊意义，除了^,-之外。

如[\d.\-+]表示数字，小数点，-，+

4. 量词

{n} 表达式重复n次
{m,n} 重复至少m次，至多n次
{m,} 重复至少m次
? 0次或1次，{0,1}
+ 至少1次，{1,}
* 不出现或出现任意次，{0,}

5. 字符边界

^ 与字符串开始的地方匹配
$ 与字符串结束的地方匹配
\b 匹配一个单词边界, (前面的字符和后面的字符不全是\w)

6. 选择符和分组

| 两个表达式之间 或 关系
() 括号内的表达式可以作为整体被修饰，()\1表示捕获的整体作为匹配模式 go gogo
(?:表达式) 抵消()带来的作用

7. 预搜索-零宽断言

(?=exp) 断言后匹配exp
[a-z]+(?:\d) go222 222本身不计入匹配
(?<=exp)< td> 断言前匹配exp
(?!exp) 断言后不匹配exp
(? 断言前不匹配exp
