Java基础知识-线程

发表于 2019-09-14 更新于 2020-05-14 分类于 Java 热度： ℃ 评论：

多线程

进程

处于运行过程中的程序。拥有独立性，动态性，并发性

并发指多个进程指令在cpu中被快速轮换宏观上实现并发

并行指同一时刻，多条指令在多个处理器上同时执行

创建线程

调用的是start方法，使其创建一个执行run方法的新线程

extends Thread

通过new Thread()方式，创造了新的实例变量，线程之间不存在资源共享

implements Runnable

线程之间资源共享

implements Callable

线程状态

  • New

    jvm分配内存，初始化成员变量值，此时的线程等同于普通类

    调用.start() 线程进入就绪状态，等待cpu的调度

  • Runnable

    争取cpu，执行run()内的代码，执行时长为时间片长

    是否运行取决于cpu

  • Blocked

    时间片用完

    发生阻塞式IO

    sleep()方法主动放弃处理器资源

    同步资源被锁定

    等待通知notify

    ….

    线程进入阻塞状态

  • Waiting

    解除了阻塞原因后，线程重新进入就绪状态

  • Timed waiting

  • Terminated

控制线程

interrupt()

请求终止线程，线程的中断状态将被置位

isInterrupted() 判断中断状态，不改变中断状态

如果线程被阻塞，无法检测中断状态

join() 插队

new Thread().join() 等待该线程执行完才执行其他线程

守护线程

为其他线程提供服务

new Thread().setDeamon(true)

jvm不会等待该线程运行结束。当整个虚拟机中只有这种线程时，虚拟机选择退出

sleep() 睡眠

Thread.sleep() 暂停当前线程进入阻塞状态，不释放资源

yield() 礼让

Thread.yield() 该线程主动让出时间片进入就绪状态

改变线程优先级

优先级也不能保证线程执行的先后顺序

new Thread().setPriority(Priority.MIN_PRORITY) 1

Priority.MAX_PRORITY 10

Priority.NORM_PRORITY 5

竞争

ReentrantLock 可重入锁

锁可以延续使用并拥有计数器

一旦线程进入就封锁锁对象，其他线程无法通过lock语句，被阻塞

synchronized 同步

可以是同步块，同步方法

一个对象里面如果有多个synchronized方法，某一个时刻内，只要一个线程去调用其中的一个synchronized方法，其他线程都只能等待，锁的是当前的this，实例对象

而类中的普通方法和同步锁无关

当有多个对象时，就不是同一把锁了，每个对象的方法可以并发运行

静态同步方法中，锁的是整个类

同步方法块中，synchronized锁的是括号里的代码

锁在退出或异常退出后必须释放锁，防止其他线程饥饿等待
